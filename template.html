Current situation

It's not dead, but the normal way to access it (messenger.hotmail.com:1863) will get you connection refused.

In theory this means MSN is now HTTPS only, but the all the internal servers seem to have port 1863 open still

Alternatives:

  • Enable http method. Most servers support this (all of them if HTTPS). (FAQ entry for pidgin)
  • Pick one working IP randomly from the lists below and tell your client to connect to it. (FAQ entry)

Also, keep your clients up to date!

Messengergeek wrote a nice blog post explaining the situation in a bit more of detail

Breaking news

2014-12-17: Two main bn1 servers closed port 80 (HTTP). HTTPS still works, but few clients use it.

2014-12-09: Two main db3 servers closed port 1863. These were the last externally exposed addresses. HTTPS connections to all these servers still work.

2014-12-04: Two main bn1 servers closed port 1863.

2014-12-01: Two main bay servers closed port 1863.

2014-11-13: ApplicationID for windows live messenger 2008 and 2009 blocked. Messengergeek blog post

Reference #

bay Server region codename (first three characters of the hostname of each of those servers)
Highlight The highlighted rows are the main servers of a region, which redirect to the rest.
:80 HTTP port open. This is what you'd use when enabling HTTP method.
:443 HTTPS port open. Same as above, but not all clients use this. Newer versions of official clients use it (including WLM 2012, the windows 8 messaging client and skype)
:1863 MSNP port open. The good old standard msn protocol port.
VER Sends VER 1 MSNP18 to port 1863 to test if the server still supports that version (the most common one among third party clients). Also helps check for false positives

FAQ #

What about all the news saying it's gone? #

That's partly because nobody does fact checking and partly because the criteria of "dead" here is more oriented to third party clients

It was supposed to die in march 2013. The only thing that changed was a forced upgrade of the official clients to skype. Before mid-november 2014 the only issue was that offline messages and mail notifications didn't work.

Right now it seems like it might be dying, so I made this to track the progress of its death.

Getting connection refused #

Try switching to http method

If your client doesn't support http method, you can try setting the server address to one of the working IPs from above

One way to set the server address is to edit your hosts file and assign it to messenger.hotmail.com by adding a line like this:

134.170.24.231 messenger.hotmail.com

Remember to change that IP - overloaded servers may redirect you back to the broken ones!

How do I switch to HTTP method in pidgin? #

  1. In the accounts window, click modify
  2. Go to the advanced tab
  3. Check "use http method"
  4. Change HTTP Method Server to one of the working IP addresses above, as pidgin doesn't connect through HTTPS

That's all. All the other fields are irrelevant and can be left as-is

Some contacts are offline #

Might be a pidgin bug that sneaked in the last release. Try removing that contact and adding it again

If everything fails, outlook.com has a webchat (using MSNP24!), any msn login will work

How do I fix my BitlBee? #

Native BitlBee: Doesn't support HTTP method (I'm working on it, okay!)

If you're using revision 1070, just do:

account msn set server 134.170.24.231

If you're using revision 1064, edit your hosts file as suggested in a previous question. If you're using something older, it won't work because of the ApplicationID block. Except, maybe, 1.x, but please don't.

bitlbee-libpurple: Enable http method like this:

account msn set http_method on

Also get libpurple 2.10.11 or it won't work because of the ApplicationID block.

Public servers: testing.bitlbee.org usually runs the latest version and should work.

See also: HowtoMsn wiki page

Why care about MSN? #

A few reasons:

  • It's a decent bridge to the skype network
  • Works in clients that aren't skype
  • I like the idea of delaying the inevitable by catching energy boosts which reverse the setting sun - if only for a moment
  • Fun things are fun

What does the future hold? #

This is speculation, but based on the current trends, it will keep breaking slowly, based on what microsoft doesn't need to keep alive anymore

The HTTP(S) method (port 443 in the status table above) is likely to stay up. These servers are still used by skype, although with a newer protocol.

The messaging client included in windows 8.0 (which they probably still need to support) uses a MSNP22 - a protocol version older than skype's, newer than most other third party clients, but using the same connection method as the older ones, through HTTPS
