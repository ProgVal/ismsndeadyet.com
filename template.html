Current situation

It's not dead, but it can only be accessed decently with http method. So just enable that

More precisely: the default connection address, messenger.hotmail.com redirects to 6 servers, which are in charge of redirecting to the rest.

Those 6 servers either have closed the MSNP port (and gone HTTP only), or do internal redirections to servers that don't work

Breaking news

What clients work

(MSNP## denotes the protocol version number)

  • Pidgin (MSNP18) and derivatives using libpurple 2.10.11. See also "Some contacts are offline" below
  • Adium patched with adium messenger reviver
  • BitlBee (MSNP18), dev versions >1064. Read more. Does not support http method, unless libpurple is installed
  • Trillian, latest beta build

Official clients:

  • Windows Live Messenger 2008-2012 (MSNP15-MSNP21) patched with messenger reviver
  • Skype and outlook.com (MSNP24), but you probably don't care about those
  • Windows messenger 4.7 (MSNP8) from windows XP still works. Seriously

FAQ

What about all the news saying it's gone?

That's partly because nobody does fact checking and partly because the criteria of "dead" here is more oriented to third party clients

It was supposed to die in march 2013. The only thing that changed was a forced upgrade of the official clients to skype. Before mid-november 2014 the only issue was that offline messages and mail notifications didn't work.

Right now it seems like it might be dying, so I made this to track the progress of its death.

Getting connection refused

Try switching to http method, or add a line to your hosts file assigning messenger.hotmail.com one of the working IPs taken from above, for example:

134.170.24.231 messenger.hotmail.com

Some contacts are offline

Might be a pidgin bug that sneaked in the last release. Try removing that contact and adding it again

If everything fails, outlook.com has a webchat (using MSNP24!), any msn login will work

Why care about MSN?

A few reasons:

  • It's a decent bridge to the skype network
  • Works in clients that aren't skype
  • I like the idea of delaying the inevitable by catching energy boosts which reverse the setting sun - if only for a moment
  • Fun things are fun

What does the future hold?

This is speculation, but based on the current trends, it will keep breaking slowly, based on what microsoft doesn't need to keep alive anymore

The HTTP(S) method (port 443 in the status table above) is likely to stay up. These servers are still used by skype, although with a newer protocol.

The messaging client included in windows 8.0 (which they probably still need to support) uses a MSNP22 - a protocol version older than skype's, newer than most other third party clients, but using the same connection method as the older ones, through HTTPS
